Implementation Path

From kickoff to production in six weeks.

Standard implementations follow this arc. Pilot is compressed (single user); Embedded extends post-week-6 indefinitely.

Week 1 Discovery

Interview the team. Catalog mail volume + types. Define category taxonomy. Identify integration points (Slack, CRM, ticketing).

Week 1–2 Infrastructure

Provision the host (cloud or on-prem). Install dependencies. Configure TLS. Set up monitoring integration.

Week 2 Provider Connection

Register OAuth apps with Google / Microsoft. Configure IMAP credentials. Set up Pub/Sub or Graph push subscriptions.

Week 3–4 Calibration

Initial classification on a backlog of historical mail. Operator-review of category assignments. Rule additions where the classifier needs reinforcement.

Week 4–6 Style Training

Sent-mail mining for writing-style profile. Initial drafted replies for operator-review. Captured-edit feedback loop established.

Week 6 Production Cutover

Push integration enabled. Real-time triage active. Daily / weekly digest schedule established. 30-day post-launch hypercare begins.

Post-week-6 Ongoing

Monthly tuning calls. Quarterly category review. Security patch + version updates. For Embedded engagements, this continues as long as the retainer is active.

Self-Service vs Service Offering

When is each right?

The product is real. Anyone with engineering chops can deploy it from the public repository at github.com/Unlimited-Data-Works-LLC/Email-Triage using the signed container image at ghcr.io/unlimited-data-works-llc/email-triage (verification recipe in the repo's docs/install.md). The service offering is for organizations where:

  • The deployment / customization / integration work isn't the customer's core competency
  • Compliance review needs an external attestation rather than self-attestation
  • Ongoing tuning + maintenance needs an SLA, not a Slack DM
  • The category taxonomy needs to be co-designed with someone who has done this for similar organizations

If your team has Linux/Podman fluency and an Ollama host already, self-hosting is straightforward. If you'd rather get to value without owning the implementation, that's what the service tiers are for.
