Use Cases

Five concrete scenarios where Email Triage delivers measurable outcomes. Each maps to a specific tier and links to the matching Fractional CTO engagement.

Standard or Embedded tier

Medical Researcher / Principal Investigator

Context: High inbox volume — sponsor mail, IRB notices, trainee updates, manuscript reviews, newsletters. HIPAA-aware by virtue of institutional environment. Time-pressed; daily inbox triage is a real tax.

Outcome: Daily digest collapses 50+ newsletter messages into one. Sponsor mail labeled and surfaced for response. IRB notices auto-filed and notified via separate channel. Manuscript-review threads tracked with custom category. Meeting-request intercept handles collaborator scheduling.

Standard tier · HIPAA-mode

Solo Practitioner Clinic (1–5 person practice)

Context: M365 or Google Workspace-based. HIPAA-regulated by default. Need to demonstrate compliance posture to insurers + auditors.

Outcome: HIPAA-mode accounts with PHI-scrubbed logs. Audit chain on every staff access. Drafted patient-correspondence replies for clinician review (never auto-sent). Delegate access for front-desk staff to triage but not draft. Quarterly compliance review packet produced from the audit log.

Embedded tier · self-use

Fractional CTO / Consulting Firm

Context: Multiple client mailboxes (separately authenticated). Need to keep client communications isolated for confidentiality. Calendar coordination across 8–12 active clients per week.

Outcome: Per-client account with isolated rules and routes. Per-client writing-style profile so responses sound right for each engagement (the assistant's "voice" matches the engagement). Meeting-request intercept on every account with the right working-hours config per client. Watch rules escalate "fire-drill" client emails to SMS instantly.

This is how I run my own consulting practice.

Standard tier · multi-user

Researcher Cohort Tooling

Context: A lab with N researchers all using shared mail infrastructure. Each researcher has their own writing style and their own preferences. Privacy: one researcher shouldn't see another's mail or drafts.

Outcome: Multi-user install with role-based access. Each researcher gets their own dashboard, their own categories, their own digest. Lab PI has delegate access to specific researcher mailboxes (with audit-row stamping). One Ollama host serves the whole lab; one container, one TLS cert, one backup.

Self-hosted · no service tier needed

Privacy-First Homelab Operator

Context: Self-hosting enthusiast running on Podman. Wants Email Triage as part of a Tailscale-funneled internal-services stack. LLM already running locally (Ollama, GPU).

Outcome: Install + configure independently from the public repository. Service offering not needed but documentation and ongoing version updates are valued. Tailscale-issued TLS certificate. Audit chain and HIPAA-mode features work the same as in enterprise deployments — privacy posture isn't tier-gated.
